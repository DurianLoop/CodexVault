# Codex Vault v0.3 Next Stage Plan

This document defines the next stage after the v0.2 local-confidence beta.

v0.2 proved that Codex Vault can behave like a trustworthy local product: it scans `.codex`, summarizes skills and memories, previews imports, supports rollback, persists achievement progress, and presents a more polished desktop UI.

v0.3 should make Codex Vault ready for a wider private beta: easier to install, safer to test, clearer to trust, and closer to real multi-device sync.

## Stage Goal

Ship a Windows-first private beta that a non-developer tester can download, run, use on a real `.codex` folder, export/import packs safely, and understand what is local-only versus sync-ready.

## Product Position

Codex Vault should remain local-first. Cloud sync is a capability, not the default behavior.

The next stage should avoid turning the app into a large social platform. It should feel like a focused migration cockpit for Codex users who care about their skills, memories, prompts, backups, and progress history.

## Non-Goals

- Do not require administrator privileges.
- Do not upload `.codex` content automatically.
- Do not make public share codes available before access control is designed.
- Do not add a marketplace or social feed.
- Do not migrate to Tauri or Electron before the Python/Tkinter beta is packageable.
- Do not add new SVG-based product illustrations or README visuals. Prefer real screenshots and PNG assets.

## Priority Overview

| Priority | Track | Outcome |
| --- | --- | --- |
| P0 | Windows packaging | Tester can run Codex Vault without installing Python manually. |
| P0 | Import confidence | User can preview, compare, import, and roll back without guessing. |
| P0 | Achievement polish | Achievement activation, filtering, progress, icons, and history feel stable. |
| P1 | Sync-ready backend | Backend API has real auth boundaries and pack ownership semantics. |
| P1 | Release documentation | README, user guide, and security notes match the actual app. |
| P2 | Visual QA automation | Repeatable screenshot checks catch broken desktop layouts. |
| P2 | Future shell evaluation | Tauri/Electron decision is based on product needs, not novelty. |

### Milestone B: Import Confidence Release

## Recommended Definition Of Done

- Full test suite passes:

```powershell
python -B -m unittest discover -s tests -v
```

- Packaged Windows build opens without Python installed.
- Import preview and rollback are tested with a temporary `.codex` fixture.
- README uses real PNG screenshots, not SVG mockups.
- The app can be used without network login.
- Backend sync paths require explicit user action.
- No secret-like files are included in exported packs by default.

## Risks And Mitigations

| Risk | Mitigation |
| --- | --- |
| PyInstaller misses image assets | Add startup asset diagnostics and packaging tests. |
| Tkinter custom chrome regresses close/minimize behavior | Keep smoke tests for window controls and view switching. |
| Sync scope grows too large | Keep v0.3 sync as backend architecture preview, not public cloud launch. |
| Import diff becomes too complex | Start with markdown text diff only; defer binary diff. |
| Users misunderstand local versus cloud behavior | Add visible local-only status and documentation warnings. |
| Antivirus flags packaged executable | Provide source-run fallback and document hash/version. |
