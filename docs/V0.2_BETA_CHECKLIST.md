# Codex Vault v0.2 Beta Checklist

This checklist records what the v0.2 local-confidence slice completed and what remains outside this beta.

## Completed

- ✅ Unified dark custom window chrome for the main app window.
- ✅ Centered custom achievement detail modal without the default white system title bar.
- ✅ Achievement cards remain clickable and reveal requirement/progress details.
- ✅ Achievement progress snapshots are persisted in `data/achievement_state.json`.
- ✅ Achievement notification history is persisted locally.
- ✅ Achievement wall shows a compact recent-unlock feed.
- ✅ Import preview is available before writing files.
- ✅ Import preview separates added, replaced, and skipped files.
- ✅ Import still creates a backup before writing.
- ✅ Share-code import uses the same preview-before-write flow.
- ✅ Backup rollback is available from the Sync / Packs screen.
- ✅ Core tests cover import preview and rollback.
- ✅ Achievement tests cover v0.2 progress and notification state shape.

## Not Completed

- ❌ Real hosted cloud accounts are not deployed.
- ❌ Public internet share codes are not available.
- ❌ PostgreSQL/object-storage production backend is not implemented.
- ❌ PyInstaller portable build is not completed.
- ❌ Tauri/Electron shell is not implemented.
- ❌ Markdown diff/merge UI is not implemented.
- ❌ Code signing is not configured.
- ❌ Full visual screenshot automation for Tkinter is not implemented.

## Current v0.2 Position

v0.2 is now a stronger local beta focused on trust, preview, rollback, and achievement polish. It is still local-first and low-dependency.
